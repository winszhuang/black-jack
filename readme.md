## 設計思路

> first version
### 客戶端主動推送
- ready(所有玩家都按下ready遊戲就開始)
- hit(玩家要牌)
- stand(玩家停牌)  

### 客戶端被動接收
- 更新玩家資訊和牌組
- 更新某玩家要牌、停牌資訊
- 更新遊戲狀態
  - 準備狀態
  - 開始遊戲
  - 遊戲結束
- 更新最終遊戲結果
